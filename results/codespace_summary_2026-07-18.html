1. What has been implemented

  • Backend review system with FastAPI, LangGraph orchestration, and SSE progress streaming.
  • Five specialized agents: security, bug_detection, style, performance, and fix.
  • Agent contract implemented in src/agents/base.py with `run(files, context)` and per-file `analyze(code, file_path, context)`.
  • Security agent supports RAG retrieval, Bandit triage fallback, and LLM security analysis.
  • Bug detection agent combines AST static analyzers for Python and LLM analysis.
  • Style agent uses `ruff` linting plus LLM style/readability findings.
  • Performance agent uses static complexity analysis plus LLM performance review.
  • Fix agent can generate patch suggestions, validate Python syntax, and commit fixes through Git service integration.
  • Evaluation harness present in eval/ with runners for PrimeVul, Defects4J, SEC-bench, OWASP/CWE, and project-agent review.
  • Persistent evaluation artifacts are written under results/.

2. Project flow

Review pipeline

  • initialize: start review state and event publishing.
  • fetch_pr: confirm PR metadata and files.
  • check_success: verify PR and files exist.
  • analyze_security, analyze_bug, analyze_style, analyze_performance run in parallel.
  • aggregate_findings: deduplicate findings and merge agent results.
  • apply_fixes conditionally runs if findings exist.
  • finalize: publish completion and finalize review status.

The orchestrator graph is defined in src/agents/orchestrator/graph.py.

3. Steps to test the entire project

  1. Install Python dependencies from the repository root: pip install -e ".[dev]".
  2. Copy .env.example to .env and populate required values for Anthropic and GitHub.
  3. Start Redis with docker compose up -d redis if using the queue or SSE backend.
  4. Run the backend: make run or python -m uvicorn src.main:app --host 0.0.0.0 --port 8000 --reload.
  5. Verify API health: curl http://localhost:8000/health and curl http://localhost:8000/ready.
  6. Start the dashboard under dashboard/: install deps and run npm run dev.
  7. Use the dashboard to submit a PR URL or call the API directly on /api/v1/reviews.
  8. Optionally configure GitHub webhook delivery to /api/v1/webhook/github for automatic review triggers.
  9. Run the evaluation harness manually from the repo root with python eval/run_evals.py.
